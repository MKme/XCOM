Short description (paste into "Product short description")

XCOM™ is an offline-first radio reference + mapping toolkit built for bad connectivity. Install it as a PWA on iOS/Android/Desktop and keep operating with no signal. It includes an offline Repeater Map (USA/Canada), Packet Stations directory, worldwide Shortwave Broadcasts schedule browser, Predict (offline callsign lookup + path tools), plus field utilities like XTOC Comm + XTOC Data (structured packets, QR, Mesh/MANET, XTOC backup import, and a tactical map Imported overlay for recent XTOC markers/zones), Meshtastic + MeshCore mesh integration, OpenMANET node mapping, a QSO Logbook, and more. Activate once with your license key while online, then keep using it offline on that device.

Long description (paste into main Product description)

What you're getting

  • Offline-ready web app (PWA): Installable on iOS/Android/Desktop. Once loaded, it keeps working without signal.
  • Local-first data: Personal lists and logs stay on your device (no accounts required).
  • Offline mapping: Set an AO and cache map tiles for that area to keep maps usable offline.
  • Operator-focused tools: Repeaters, packet nodes, shortwave schedules, callsign lookup, logging, and more.

Key modules

  • Predict: offline callsign lookup (USA/Canada), path plotting, and a quick HF prediction tool.
  • Repeaters: find repeaters by radius/band/mode and view tones, offsets, notes, and distance.
  • Packet Stations: map packet nodes/BBS and keep a personal local list.
  • Shortwave Broadcasts: browse worldwide HF schedules by time ("what's on now") or by frequency.
  • Map: XTOC-style tactical map + offline raster tile caching, plus Mesh nodes (Meshtastic / MeshCore / OpenMANET) with optional node-to-roster assignment, and an Imported overlay for XTOC packet markers/zones (type toggles; last 7 days rendered by default). Select a marker and choose Hide from map to declutter.
  • XTOC Comm: create/import structured packets (CLEAR or SECURE). Source Unit(s) supports multi-select for SITREP/TASK/CONTACT/RESOURCE/ASSET/ZONE, plus squad/group CHECKIN/LOC (multi-unit). Chunk for tight message limits, move by copy/paste/QR/mesh/MANET (LAN), and import XTOC backups (roster + keys + packets) into XCOM.
  • XTOC Data: local-first packet archive — browse/search all stored XTOC packets (including non-location) and import geo packets to the Map overlay.
  • Mesh: connect to a Meshtastic or MeshCore device (Web Bluetooth) and send/receive messages.
  • Logbook: fast QSO logging with ADIF/CSV export.

License & activation

  • One-time activation: enter your license key once while online.
  • Offline afterward: after activation, XCOM™ continues to work offline on that device.

Getting started (quick)

  1. Install for offline use:
    • iPhone/iPad: Safari → Share → Add to Home Screen
    • Android: Chrome menu → Install app
    • Desktop: Chrome/Edge install icon → Install
  2. Cache maps for your AO: open Map and download tiles for your operating area while online.
  3. Operate: use Repeaters/Packet/Shortwave/Predict as offline references, and log QSOs as needed.

Note: public tile servers have usage policies. For serious/off-grid use, point XCOM™ at your own tile server.

Who XCOM™ is for

  • Shortwave listeners and HF monitoring
  • Emergency preparedness and field operations
  • Off-grid comms enthusiasts using mesh/radio workflows

FAQ

Does XCOM™ require the internet?
No. Most tools are designed to run offline after installation/first load. Some features (like caching map tiles) must be done while online.

Does it require accounts?
No. XCOM™ is designed to be local-first and account-free.

Which browsers are recommended?
Modern Chrome/Edge/Safari. Mesh features typically require Chrome/Edge/Android due to Web Bluetooth support.

Does XCOM™ work with Meshtastic?
Yes. XCOM™ supports Meshtastic workflows including mapping nodes, messaging, and mesh-friendly packet transfer paths.

Does XCOM™ support MeshCore?
Yes. XCOM™ supports MeshCore workflows including mapping nodes, messaging (Web Bluetooth), and mesh-friendly packet transfer paths.
