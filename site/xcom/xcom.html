Offline-first radio reference + mapping toolkit

What XCOM™ is

XCOM™ is an offline-capable Progressive Web App (PWA) for ham radio and shortwave reference, mapping, and field workflow tools.
Use it to (examples):
  • Find repeaters near a location and see tones, offsets, modes, and notes on a map.
  • Look up a callsign offline (USA/Canada) and plot a path with a quick HF prediction.
  • Browse worldwide shortwave broadcasts by time ("what's on now") or by frequency.
  • Reference packet radio nodes/BBS stations and keep a personal local list.
  • Download map tiles for your Area of Operations so maps keep working when you go offline.

Built for bad connectivity

XCOM™ is local-first: your data stays on your device, and the app keeps working with no signal once it's loaded. It's designed for field use, travel, emergency preparedness, and radio operations where the network isn't guaranteed.
  • No accounts. No login, no cloud dependency, no server required for the core app.
  • Offline maps. Cache raster tiles for your AO while online, then use them offline later.
  • Offline location search. Look up cities/QTHs from a local dataset (no live geocoding API required).

What's inside

XCOM™ includes purpose-built modules for radio operators:
  • Predict: offline callsign lookup (USA/Canada), path plotting, and a quick HF prediction tool.
  • Shortwave Broadcasts: browse HF schedules by time or frequency (with Local/UTC views).
  • Packet Stations: map of packet nodes/BBS + frequency reference + your own local entries.
  • Repeaters: repeater map with filters (band/mode/radius) and detailed station info.
  • Map: set your AO and download offline raster tiles for that area.
  • XTOC Comm: create/import structured packets (CLEAR or SECURE), chunk them for tight limits, and move them via copy/paste, QR, or Meshtastic.
  • Mesh: connect to a Meshtastic radio (Web Bluetooth), view nodes on a map, and send/receive messages.
  • Logbook: fast QSO logging with local storage and ADIF/CSV export.
  • ASCII: generate clean copy/paste ASCII banners for packets, BBS posts, and notes.
  • Ham Clock: a quick time + operating reference view.

Quick start (offline)

1) Install as an app (recommended)

  • iPhone/iPad: Safari → Share → Add to Home Screen
  • Android: Chrome menu → Install app
  • Desktop: Chrome/Edge install icon → Install

2) Cache offline maps for your AO

  • Open Map and select Offline Raster (cached).
  • Pan/zoom to your Area of Operations.
  • Download tiles for that AO while you have internet.

Note: public tile servers have usage policies. For serious/off-grid use, point XCOM™ at your own tile server.

Compliance notes (important)

XCOM™ includes tools that can create SECURE (encrypted) payloads for some workflows. You are responsible for following local laws and radio regulations.
Ham note: do not transmit encrypted/obscured content on amateur radio. Use CLEAR mode on ham where appropriate.

Privacy & data

  • Local-first: your personal lists and log entries live in your browser storage on this device.
  • No tracking: XCOM™ is designed to run without accounts or analytics.
  • Offline friendly: most features work without internet once installed/loaded.

FAQ

Does XCOM™ require a server?
No. The core app is a static PWA and runs locally in your browser.

Do I need a license key?
On first run (when hosted), XCOM™ may prompt for a license key. Activate once while online, then keep using it offline on that device.

Will it work offline?
Yes. Install it as a PWA, and it will keep working offline after it's loaded. Offline map tiles must be cached while online.

Which browsers are recommended?
Modern Chrome/Edge/Safari. The Mesh module uses Web Bluetooth (typically Chrome/Edge/Android).

Are frequencies guaranteed accurate?
Treat any directory as a reference. Always listen/verify on-air and follow your local regulations.
